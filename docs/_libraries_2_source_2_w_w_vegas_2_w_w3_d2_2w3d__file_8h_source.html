53NAMING CONVENTIONS:
54
55 Typical render object name is 15 characters + NULL
56 Meshes have 31 + NULL character name formed from the concatenation of the "container"
57 model name and the mesh's name: "ContainerName.MeshName"
58 Animations have 31 + NULL character names formed from the concatenation of the Hierarchy tree
59 name with the animation name: "AnimationName.HierarchyName"
60 Textures have unlimited name length.
61 Typically you can determine which 'W3D' file a render object came from by looking
62 at its name. If the name contains a '.' then the filename is the string before
63 the '.' and if not, then the render object name is the name of the file.

185 Updating the mesh format as part of the conversion to Surrender1.40. Some
186 of the new features in this mesh format are:
187 - per pass, per stage U-V coordinate arrays
188 - per pass vertex diffuse color arrays
189 - per pass vertex specular color arrays
190 - per pass vertex pre-calced diffuse illumination arrays
191
192 In addition, the way you describe the materials for a mesh has *completely*
193 changed. The new system separates the concepts of VertexMaterial, Texture
194 and Shader. A VertexMaterial defines the parameters which control the
195 gradient calculations for vertices. Textures you know about. Shaders
196 define how the gradients (diffuse and specular) are combined with the texture
197 and the frame buffer. In addition, a mesh can have several passes; each
198 pass having VertexMaterials, Textures and Shaders
199 - new chunk to describe a shader, contains a W3dShaderStruct
200 - new chunk to describe a vertex material, contains a W3dVertexMaterialStruct
201 - textures use the old 'Map3' chunk still
202 - new chunk to describe a "pass" for the mesh
203 - new chunk to tie up all of these vertex materials, shaders, and textures
204 - new chunks for specifying per-poly, per-pass arrays of texture, shader, and
205 vertex material indices.

1317/********************************************************************************
1318
1319 WHT ( Westwood Hierarchy Tree )
1320
1321 A hierarchy tree defines a set of coordinate systems which are connected
1322 hierarchically. The header defines the name, number of pivots, etc.
1323 The pivots chunk will contain a W3dPivotStructs for each node in the
1324 tree.
1325
1326 The W3dPivotFixupStruct contains a transform for each MAX coordinate
1327 system and our version of that same coordinate system (bone). It is
1328 needed when the user exports the base pose using "Translation Only".
1329 These are the matrices which go from the MAX rotated coordinate systems
1330 to a system which is unrotated in the base pose. These transformations
1331 are needed when exporting a hierarchy animation with the given hierarchy
1332 tree file.
1333
1334 Another explanation of these kludgy "fixup" matrices:
1335
1336 What are the "fixup" matrices? These are the transforms which
1337 were applied to the base pose when the user wanted to force the
1338 base pose to use only matrices with certain properties. For
1339 example, if we wanted the base pose to use translations only,
1340 the fixup transform for each node is a transform which when
1341 multiplied by the real node's world transform, yeilds a pure
1342 translation matrix. Fixup matrices are used in the mesh
1343 exporter since all vertices must be transformed by their inverses
1344 in order to make things work. They also show up in the animation
1345 exporter because they are needed to make the animation work with
1346 the new base pose.
1347
1348********************************************************************************/

1516/********************************************************************************

1518 HMorphAnimClass
1519
1520 This is an animation format which describes morphs between poses in another
1521 animation. It is used for Renegade's facial animation system. There is
1522 a normal anim which defines the pose for each phoneme and then a "Morph Anim"
